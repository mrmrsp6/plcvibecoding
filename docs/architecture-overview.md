# 高层架构概述

本文档从用户视角描述 PLCvibeCoding 的宏观架构，不涉及离线解析链路、内部路由框架等实现细节。

## 技术栈

- **桌面端**：Electron + React + TypeScript
- **UI 框架**：React 函数组件 + 状态管理
- **AI 后端适配**：统一接口层，支持 OpenAI / Anthropic / DeepSeek / Ollama 等多种模型
- **打包**：electron-builder，输出 Windows 安装包与便携版

## 分层视图

```text
┌─────────────────────────────────────────────┐
│  用户界面层  │  连续梯形图工作区 / AI 协作面板 / 工程树  │
├─────────────────────────────────────────────┤
│  应用服务层  │  工程导入 / 视图渲染 / Patch 预览 / 写回控制  │
├─────────────────────────────────────────────┤
│  品牌适配层  │  三菱链路 / 西门子链路（抽象接口，具体实现闭源） │
├─────────────────────────────────────────────┤
│  AI 适配层   │  多模型统一调用 / Prompt 护栏 / 技能召回      │
├─────────────────────────────────────────────┤
│  桌面运行时  │  Electron 主进程 / 文件系统 / 加密缓存        │
└─────────────────────────────────────────────┘
```

## 关键设计原则

1. **预览优先**：任何写回操作都必须先经过可视化预览与人工确认。
2. **品牌隔离**：三菱与西门子链路通过统一抽象接口接入，互不污染。
3. **上下文隔离**：不同工程项目的会话、缓存、配置相互隔离。
4. **可追溯**：每一次 AI 协作与补丁操作都留有审计线索。

## 安全边界

- AI 不直接修改现场 PLC 或原始工程文件。
- 写回需要显式授权。
- 项目级缓存采用加密存储。

## 开源与闭源边界

- **本仓库公开**：产品文档、通用 PLC 协作技能、脱敏示例工程。
- **闭源**：桌面应用源码、品牌适配实现、工程解析与写回细节。
